Getting Started

kouji-ui is a headless Angular 21 UI library. The core has zero CSS — bring your own styles or use the Tailwind v4 implementation package.

Installation

Install the headless core for custom styling:

$ pnpm add @kouji-ui/core

Or install the Tailwind v4 styled implementation:

$ pnpm add @kouji-ui/ui
◈

Requires Angular 21. The library uses signal inputs, afterNextRender, and inject() — all Angular 21+ features.

Quick start

Import directive directly — no NgModules needed.

import { KjButtonDirective } from '@kouji-ui/core';

@Component({
  standalone: true,
  imports: [KjButtonDirective],
  template: `
    <button kjButton [kjVariant]="'destructive'">
      Delete
    </button>
  `,
})

Headless usage

The core package ships zero CSS. Directives add ARIA attributes and data-* attributes you can target in your stylesheets:

/* Style via data attributes — works with any CSS approach */
[kjButton] {
  padding: 0.5rem 1rem;
  background: var(--color-primary);
}

[kjButton][data-variant="destructive"] {
  background: var(--color-danger);
}

[kjButton][aria-disabled="true"] {
  opacity: 0.5;
  cursor: not-allowed;
}

With Tailwind v4

Import @kouji-ui/ui's stylesheet to get the pre-built Tailwind v4 component styles:

/* In your global styles.css */
@import "@kouji-ui/ui/styles/kj.css";

Then use styled components from @kouji-ui/ui:

import { KjButton } from '@kouji-ui/ui';

// KjButton is a pre-styled Angular component wrapping KjButtonDirective

Angular forms

Form controls compose KjFormControlDirective via hostDirectives, so they work with reactive forms out of the box:

import { KjInputDirective } from '@kouji-ui/core';
import { ReactiveFormsModule, FormControl } from '@angular/forms';

@Component({
  standalone: true,
  imports: [KjInputDirective, ReactiveFormsModule],
  template: `
    <input kjInput [formControl]="email" type="email" />
    <span *ngIf="email.invalid && email.touched">
      Invalid email
    </span>
  `,
})
export class MyForm {
  email = new FormControl('');
}

Accessibility

Every component targets WCAG 2.1 AAA. Keyboard interactions, ARIA patterns, and focus management are handled by the CDK primitives under the hood.

Use the a11y utilities to enhance your own components:

import {
  KjFocusTrapDirective,
  KjLiveRegionDirective,
  KjRovingTabindexDirective,
  KjVisuallyHiddenDirective,
} from '@kouji-ui/core/a11y';
♿

All overlay components (Dialog, Menu, Select) use Angular CDK's FocusTrap, FocusKeyManager, and LiveAnnouncer for full keyboard and screen reader support.
