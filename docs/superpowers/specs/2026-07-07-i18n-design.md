# i18n — Typed Translation Strings — Design Spec

**Date:** 2026-07-07
**Status:** Approved
**Branch:** `feat/i18n` (stacked on `feat/locale` / PR #15)
**Roadmap:** `apps/docs/src/app/pages/roadmap/items/v0.2-i18n.md`

## Goal

Make every **visible / assistive-text** string the library hard-codes in
TypeScript replaceable per locale — toast close, dialog dismiss, pagination
labels ("Page N of M"), and a11y live-region announcements. Strings live in
**typed, tree-shakable per-locale catalogs**; the active catalog is selected
from the locale that already exists — `KjLocale.locale()` (PR #15) — with a
guaranteed fallback to a default English (`en`) catalog so the UI never blanks
out on a missing key.

This is the **words themselves** layer. It sits *on top of* the locale provider
(which decides how *data* renders — numbers, currency, dates, direction). i18n
reads `KjLocale.locale()` as its selector and never re-implements locale state.

## Non-goals (deferred — YAGNI)

- **Translating every string in every component.** MVP ships the service +
  directive + a typed default catalog covering the *named* strings, wires a few
  real ones (toast close for real; pagination / dialog / a11y keys defined and
  demoed), and one alternate locale (`fr`) as proof. Retrofitting each
  component's static config token (e.g. `KJ_PAGINATION_CONFIG`) to source from
  the catalog is a follow-up.
- **Full ICU message format** (`{count, plural, one {#…} other {#…}}`, gender,
  nested select). MVP does named `{placeholder}` interpolation only. ICU
  pluralisation is a documented follow-up — the API surface (`params` object) is
  forward-compatible with it.
- **A rendered language-switch UI.** Runtime locale changes go through
  `KjLocale.setLocale()` (PR #15). The docs example includes a switch button to
  *demonstrate* reactivity; a shipped `KjLanguageMenu` is a separate ticket.
- **Lazy/async catalog loading over the network.** Catalogs are plain
  `import`-able modules registered at provider time; code-splitting is left to
  the consumer's bundler (they are tree-shakable, so unused locales never ship).

## Public API

Exported from `@kouji-ui/core` (folder `packages/core/src/i18n/`).

### Typed keys & catalog shape

The `en` catalog is the **source of truth**; the key union is derived from it, so
every key that exists is spelled exactly once and typos are compile errors.

```ts
export const EN_CATALOG = {
  'toast.close':          'Close notification',
  'dialog.close':         'Close dialog',
  'pagination.nav':       'Pagination',
  'pagination.previous':  'Previous page',
  'pagination.next':      'Next page',
  'pagination.first':     'First page',
  'pagination.last':      'Last page',
  'pagination.more':      'More pages',
  'pagination.page':      'Page {page}',
  'pagination.pageOf':    'Page {page} of {total}',
  'a11y.pageChanged':     'Page {page} of {total}',
  'a11y.selected':        'Selected',
  'a11y.sortApplied':     'Sort applied, {rows} rows',
} as const;

export type KjTranslationKey = keyof typeof EN_CATALOG;
export type KjTranslationCatalog = Record<KjTranslationKey, string>;
export type KjTranslationParams = Record<string, string | number>;
```

Alternate catalogs are `Partial<KjTranslationCatalog>` — a translator may omit
keys and each missing one falls through to `en` (roadmap: "missing keys fall
through to the source language"). Because the key type is derived from `en`, a
misspelled key in `fr.ts` fails `tsc`.

### `provideKjTranslations(catalogs): EnvironmentProviders`

Registers alternate catalogs, keyed by BCP-47 tag or bare language subtag.
Multi-provider (`KJ_TRANSLATION_CATALOGS`), so several calls compose. `en` is
always present without registration. Tree-shakable: only the catalog modules a
consumer *imports* are bundled.

```ts
import { FR_CATALOG, provideKjTranslations } from '@kouji-ui/core';

bootstrapApplication(App, {
  providers: [
    provideKjLocale({ locale: 'fr-FR' }),   // PR #15 — selects the catalog
    provideKjTranslations({ fr: FR_CATALOG }),
  ],
});
```

### `KjTranslateService` (`providedIn: 'root'`)

Keeps the `Service` suffix because the primary directive is named `KjTranslate`
(same rule as `KjToastService` / `KjToast`).

```ts
class KjTranslateService {
  /** Imperative lookup. Reads KjLocale.locale() — reactive inside computed/effect. */
  translate(key: KjTranslationKey, params?: KjTranslationParams): string;
  /** Reactive signal wrapper for the same lookup. */
  translation(key: KjTranslationKey, params?: KjTranslationParams): Signal<string>;
  /** Register a catalog at runtime (mirrors provider-time registration). */
  register(locale: string, catalog: Partial<KjTranslationCatalog>): void;
}
```

**Catalog selection** for `translate(key)`:

1. Read `KjLocale.locale()` (e.g. `'fr-FR'`).
2. Exact tag match (`fr-FR`) → bare language subtag (`fr`) → `en`.
3. Within the chosen catalog: key present? use it. Else fall through to `en`'s
   value. Else return the key string itself (never blank).
4. Interpolate `{name}` tokens from `params`.

### `KjTranslate` directive (primary)

Signals-first, directive-led (no CDK). Writes the translation into the host —
either its text content or a named attribute (for `aria-label`, `title`, …).
Fully reactive: re-renders when the locale or params change.

```ts
@Directive({ selector: '[kjTranslate]' })
class KjTranslate {
  kjTranslate = input.required<KjTranslationKey>();
  kjTranslateParams = input<KjTranslationParams>();
  /** Target attribute (e.g. 'aria-label'); default writes textContent. */
  kjTranslateAttr = input<string>();
}
```

```html
<!-- visible text -->
<span [kjTranslate]="'pagination.pageOf'"
      [kjTranslateParams]="{ page: 3, total: 12 }"></span>
<!-- localized aria-label on an icon button -->
<button [kjTranslate]="'toast.close'" kjTranslateAttr="aria-label">×</button>
```

A pipe is intentionally omitted for MVP: a pure pipe would not react to the
locale signal and an impure pipe re-runs every change-detection tick. The
directive + `translation()` signal cover template and programmatic use cleanly.

## What ships in THIS PR (MVP)

- `KjTranslateService`, `KjTranslate` directive, `provideKjTranslations`, typed
  `EN_CATALOG` / key union, and `FR_CATALOG` (French) as proof — all exported.
- **Real component wiring:** `KjToastClose` gains a localized default
  `aria-label` from `'toast.close'` (overridable via a new `kjToastCloseLabel`
  input) — reactive to the locale.
- **Catalog coverage** for the named strings (dialog close, pagination labels,
  a11y announcements) and a **docs example** that wires them live — `[kjTranslate]`
  visible text + `aria-label` + a live-region announcement + a locale-switch
  button — proving locale reactivity end-to-end.
- Unit tests (lookup, interpolation, locale switch, subtag resolution,
  missing-key + unknown-key fallback) and a Playwright E2E (switch locale →
  assert a visible + aria string changes).

## Deferred (follow-ups)

- Retrofit each component's static config token (pagination, breadcrumb, dialog,
  table announcements) to read from the catalog.
- ICU plural / select formatting.
- Shipped `KjLanguageMenu` switch UI and async catalog loading.

## Why this shape

- **Selector = `KjLocale.locale()`** — one locale source of truth; a language
  switch already exists as `setLocale`, so i18n reactivity is free.
- **Key union derived from `en`** — typed keys with zero duplication; alternates
  are `Partial`, giving safe fallthrough *and* compile-checked spelling.
- **Directive + signal, no pipe** — matches the library's signals/standalone,
  no-CDK style and avoids impure-pipe change-detection cost.
- **`EnvironmentProviders` + `@doc`** — mirrors `provideKjLocale` / `provideIcons`,
  auto-surfaces in docs, app/route-scoped.

## Accessibility (WCAG 2.1 AAA)

- Localized `aria-label`s (toast close) and localized **live-region
  announcements** mean assistive tech reads copy in the user's language
  (3.1.2 Language of Parts; 4.1.2 Name, Role, Value; 4.1.3 Status Messages).
- Visible text and its `aria-label`/`aria-valuetext` come from the *same* catalog
  key, so they never disagree (1.3.1 Info and Relationships).
- The directive only writes text/attributes — it introduces no focusable UI and
  no new keyboard surface; focus order and contrast are unaffected.
- App authors remain responsible for the document `lang` attribute; the spec
  notes it — catalog selection follows `KjLocale.locale()`, and consumers should
  keep `<html lang>` in sync when they call `setLocale`.
